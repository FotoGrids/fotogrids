# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

Only the latest release of FotoGrids receives security updates. Please keep the
plugin up to date.

## Reporting a Vulnerability

We take the security of FotoGrids seriously. If you discover a security
vulnerability, please report it privately so we can fix it before it is
disclosed publicly.

**Please do not open a public GitHub issue, pull request, or support-forum post
for security reports.**

Instead, email **security@fotogrids.com** with:

- A description of the vulnerability and its potential impact
- Steps to reproduce, ideally with a proof of concept
- The FotoGrids version, plus your WordPress and PHP versions
- Any suggested remediation, if you have one

What to expect:

- An acknowledgement within **3 business days**
- An assessment and, for confirmed issues, a target timeline for a fix
- Credit in the release notes once the issue is resolved, if you would like it

Please give us a reasonable opportunity to release a fix before any public
disclosure. We appreciate responsible disclosure and will work with you in good
faith.

## Scope

This policy covers the **FotoGrids (Free)** plugin in this repository. The
bundled Freemius SDK under `src/freemius/` is third-party code; vulnerabilities
in the SDK itself should also be reported to
[Freemius](https://freemius.com/).
